# Sprint Documents

Sprint planning documents for the fluid simulator project.

## Naming Convention

`SPRINT-NNN.md` where NNN is a zero-padded three-digit number.

## Template

Each sprint document should include:
- Overview (why and high-level approach)
- Use Cases
- Architecture (diagrams, components, data flow)
- Implementation Plan (phased with files and tasks)
- Files Summary (table of files, actions, purposes)
- Definition of Done
- Risks & Mitigations
- Dependencies
- Open Questions
